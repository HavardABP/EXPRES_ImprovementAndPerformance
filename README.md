## About This Repository

This repository is the central workspace for the **EXPRES Improvement & Performance (I&P) team**, focused on developing and maintaining Power BI solutions using an **agentic development workflow**.

The goal is to leverage AI-assisted (agentic) tooling to accelerate Power BI development, improve consistency, and enable version-controlled, collaborative report building using the `.pbip` format.

---

## What Is Agentic Power BI Development?

Agentic Power BI development uses AI coding agents (such as GitHub Copilot) to generate, modify, and maintain Power BI project files programmatically. This is made possible by the **Power BI Project (`.pbip`) format**, which stores report and semantic model definitions as plain text files — making them readable and editable by both humans and AI agents.

### Key concepts

- **`.pbip` format** — A folder-based Power BI project format where reports and datasets are stored as structured plain text (JSON/TMDL), rather than a single binary `.pbix` file.
- **Version control** — Because `.pbip` files are text-based, they can be tracked in Git, enabling diffs, code review, and rollback.
- **AI-assisted authoring** — Agents can read, modify, and generate DAX measures, report layouts, and data model definitions directly in the file system.
- **Repeatability** — Changes made by agents are consistent, auditable, and can be templated across multiple reports.

### Typical agentic workflow

1. An agent is given a task (e.g. *"add a YTD measure to the sales table"*).
2. The agent reads the relevant `.pbip` files in the repository.
3. It modifies or creates the appropriate definition files (e.g. `model.bim`, TMDL files, or report JSON).
4. Changes are committed to a feature branch and reviewed via pull request before merging.

## Contributing Guidelines

- **Team members only** — Only members of the EXPRES I&P team should commit to this repository.
- **Use feature branches** — Never commit directly to `main`. Create a branch for your work and open a pull request.
- **Pull request reviews** — All changes must be reviewed and approved before merging.
- **Commit messages** — Write clear, descriptive commit messages that explain *what* was changed and *why*.
- **Agent-generated changes** — If a change was generated by an AI agent, note this in the pull request description and verify the output before merging.
